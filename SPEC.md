# SmartWords ‚Äî Spec

Homescreen & lockscreen widgets that show a dictionary word, its part of speech,
and its meaning. Native on both platforms, built to sip battery.

## Goal

Surface one interesting word at a time. It changes a few times a day. Reading it
costs the user a glance and costs the device almost nothing.

## Scope

| Platform | Homescreen | Lockscreen | Stack |
|----------|-----------|------------|-------|
| iOS 16+  | yes       | yes (accessory widgets) | Swift, WidgetKit, SwiftUI |
| Android 12+ | yes    | **no** ‚Äî Android has no general lockscreen widgets on phones (removed in Android 5; only partial on some tablets) | Kotlin, Jetpack Glance |

No companion app screens in v1 beyond what's required to host the widget and the
background fetch. A settings screen is a later milestone, not now.

## Architecture

The widget never makes a network call. Widgets get tiny, throttled execution
budgets on both platforms, so the pattern is:


1. A background job runs ~once/day, fetches the day's word batch (a small static
   JSON), and writes it to shared local storage.
2. The widget reads the cached batch and picks the word for the current rotation
   slot ‚Äî pure function of the clock, no I/O.
3. If the fetch fails or hasn't run yet, the widget falls back to the bundled
   seed list. It is never blank.

Result: **one network call per day**, zero during rotation.

## Data

### Source

A static JSON file per day, served from any static host (CDN, object storage,
GitHub Pages ‚Äî no server to run). The app fetches today's file by date:

```
https://<host>/words/2026-06-24.json
```

Curating words is then just publishing a JSON file ‚Äî no app update required.
**Hosting is deferred** for the proof of concept ‚Äî the bundled seed runs the
whole app offline until a host exists.

### Seed (bundled fallback)

`shared/seed-words.json` (~173 KB, 831 words) ships inside both apps. Used on
first launch, when offline, or before the first successful fetch. Same shape as
the batch minus the date.

Generated by `scripts/scrape_seed.py` from Wayne State's
[Word Warriors](https://wordwarriors.wayne.edu/list) list (word + definition +
example sentence). That source has **no part of speech**, so seed entries omit
`pos` ‚Äî see field rules. Re-run the script to refresh; it self-checks the parse.

### Contract

Daily batch (`shared/daily-batch.example.json`):

```json
{
  "date": "2026-06-24",
  "words": [
    { "word": "ubiquitous", "pos": "adjective",
      "definition": "present, appearing, or found everywhere.",
      "example": "smartphones are now ubiquitous." }
  ]
}
```

Seed (`shared/seed-words.json`): `{ "words": [ ... ] }` ‚Äî same word objects, no
`date`. One parser handles both.

Field rules:
- `word` ‚Äî lowercase headword, required.
- `definition` ‚Äî one sentence, required. Keep it short; small widgets truncate.
- `pos` ‚Äî part of speech, **optional** (`noun` / `verb` / `adjective` / ...).
  The scraped seed has none; backfill it with `scripts/fill_pos.py`
  (dictionaryapi.dev, resumable).
- `example` ‚Äî usage sentence, **optional**. Present in the seed; shown on the
  medium widget.

## Rotation

Default rotation interval: **3 hours** ‚Üí 8 slots/day. With `N` words in the
batch, the displayed word is:

```
slot  = hour_of_day / interval_hours        // 0..7 for a 3h interval
index = slot % N
```

A pure function of the clock and the cached batch ‚Äî deterministic, no state,
identical logic on both platforms.

- **iOS**: the `TimelineProvider` emits one entry per slot for the day; WidgetKit
  renders each at its scheduled time. The daily `BGAppRefresh` task refreshes the
  cache and reloads the timeline.
- **Android**: a single periodic `CoroutineWorker` (interval clamped to the
  WorkManager 15-min floor) ensures today's batch is fetched, computes the
  current slot's word, and pushes it to the widget.

## Content & layout

Shown: **word**, **definition**, **part of speech** (when present), and the
**example sentence** on the medium widget. Since the seed has no `pos`, the pos
line simply doesn't render until POS data exists ‚Äî the layout degrades cleanly.
Not in v1: pronunciation. (Easy to add later ‚Äî the contract just gains a field.)

Widget families:

| iOS family            | Use |
|-----------------------|-----|
| `.systemSmall`        | word + pos, definition truncated |
| `.systemMedium`       | word + pos + full short definition |
| `.accessoryRectangular` | lockscreen: word + truncated definition |

| Android size | Use |
|--------------|-----|
| ~2√ó2         | word + pos |
| ~4√ó2         | word + pos + definition |

## Design

Editorial / dictionary-page feel.

- **Serif comes free, natively** ‚Äî iOS `Font.system(design: .serif)` (New York),
  Android Glance `FontFamily.Serif` (Noto Serif). No font files to bundle, no
  licensing.
- Headword in serif display weight; `pos` in italic small caps feel; definition
  in regular serif.
- Light/dark via semantic system colors (no hardcoded palette).
- Lockscreen (iOS) is monochrome/tinted by the OS ‚Äî layout must read with no
  color and at tiny size, so the headword leads and the definition truncates
  gracefully.

## Refresh budget ‚Äî what the OS actually allows

- **iOS**: WidgetKit decides when to render timeline entries; background refresh
  is best-effort, not guaranteed to the minute. Our 3h rotation is well within
  what timelines support. `BGAppRefresh` for the daily fetch is also best-effort ‚Äî
  the seed/last-good cache covers any miss.
- **Android**: WorkManager periodic minimum is 15 min; our 3h cadence sits far
  above it, so updates are reliable and cheap.

Both designs degrade safely: a missed refresh just means the previous word stays
up a bit longer.

## Tech stack

| | iOS | Android |
|-|-----|---------|
| Language | Swift | Kotlin |
| Widget | WidgetKit + SwiftUI | Jetpack Glance |
| Background | BGTaskScheduler (BGAppRefresh) | WorkManager |
| Shared storage | App Group `UserDefaults` | Jetpack DataStore |
| Networking | `URLSession` | `HttpURLConnection` (stdlib) |
| JSON | `Codable` | `org.json` (bundled) |

No third-party dependencies in v1. Each can be added if a real need shows up.

## Open questions (decide before/at v1)

1. **Host for the daily JSON** ‚Äî CDN? GitHub Pages? Object storage? (Affects only
   a URL constant.)
2. **Who curates the word list** and how far ahead are batches published?
3. **Batch size per day** ‚Äî how many words should a day rotate through? (Sets `N`.)
4. **iOS deep-tap behavior** ‚Äî tap opens the app? Opens the word elsewhere?
   (Default: opens app.)
5. **Bundle IDs / package names / signing identities.**
